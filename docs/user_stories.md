Below is a list of User Stories that our team worked on, split by each team member. The number at the start of each list item is the number of story points allocated to each item.

### Alex

* **(2)** As a user, I want to update my current location so that my messages reflect my accurate position.
* **(3)** As a user, I want to set my location manually so that the system can provide location-based features when JavaScript is disabled.
* **(2)** As the system, I will map messages to location coordinates so that distance-based features work correctly.
* **(3)** As a user, I would like the app to automatically determine my location so that I have access to correct location-based features immediately without having to think about it.

### Harsh

* **(2)** As a developer, I want to deploy my Rails application from GitHub to Heroku so that I can create production-ready releases quickly and reliably.
* **(3)** As an admin, I can view all registered users with their email, display name, creation date, message count, and role, so that I can monitor and manage the user base.
* **(2)** As a user with admin role, I can access admin-only features (user and message dashboards) automatically based on my role assignment, so that access control is managed consistently rather than through email-based lists.
* **(3)** As an admin, I can view all messages with timestamps, authors, distances, and delete inappropriate messages, so that I can moderate content and maintain chat quality.
* **(1)** As a chat user, I can see messages displayed cleanly without extra whitespace, with improved CSS styling for better readability and a modern appearance, so that the chat interface is clean and visually appealing.

### Shivam

* **(2)** As a user, I want to send text messages to the global chat so that I can communicate with nearby users.
* **(3)** As a user, I want to view messages from other users so that I can participate in conversations.
* **(2)** As the system, I store each message with timestamp and location so that filtering and distance calculations are possible.
* **(2)** As a user, I want to refresh easily to see new messages so that I stay updated.
* **(2)** As a user, I want to view timestamps and sender distances so that context is clear.

### Tasnia

* **(3)** As a user, I want to sign up using email and password so that I can create a personal account.
* **(2)** As a user, I want to log in and log out securely so that my account remains protected.
* **(2)** As a admin, I will have a visibility radius set (500m) so that messages remain locally relevant.
* **(2)** As a system, I calculate distances using stored coordinates so that filtering works accurately.
* **(2)** As a user, I only see messages from users within that radius so that the chat stays location-focused.
